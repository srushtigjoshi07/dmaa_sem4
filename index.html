Title

"Hybrid Telemetry Harvester: A SQL–MongoDB Production Monitoring System for Industrial Analytics and Downtime Prediction"

Abstract

This project presents a dual-database architecture for production monitoring. Structured machine production data (shift outputs, good/scrap units, pressure metrics) is stored in a relational SQL database, while real-time sensor fault logs and schemaless telemetry drift signals are ingested into MongoDB. A web-based dashboard visualizes key performance indicators (KPIs), live sensor telemetry, and downtime forecasts. The system enables industrial analytics, active alarm tracking, and breakdown risk assessment, achieving a Global OEE Index of 98.52% in the deployed prototype.

Problem Statement

Modern industrial environments generate heterogeneous data: structured shift reports and semi-structured sensor fault logs. Storing both in a single database type leads to inefficiency—relational systems struggle with variable schemas (fault signatures), while document stores lack robust aggregation for production metrics.

Algorithm
  1. SCADA Data Capture — Read sensor triple (CoreTemp, VibNoise, OilPressure) and shift values
  2. Structured SQL Insert — INSERT INTO production_history (good_units, piston_pressure_psi, scrap_units) VALUES (...);
  3. Semi-structured MongoDB Insert — Construct BSON document → Insert into fault_logs
  4. OEE Index Calculation — OEE = (Total Good) / (Total Good + Total Scrap) × 100
  5. Breakdown Risk Forecast — Risk% = f(fault_frequency, scrap_trend, pressure_variance) → 37%
  6. Cutoff Window — MTBF extrapolation → ~140 hours
  7. Active Alarm Monitor — Count where active_alarm == true
  8. Dashboard Refresh — Update KPI cards, tables, visualizations
